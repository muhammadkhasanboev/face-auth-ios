# Entitlements & Associated Domains

## Overview

The `TadSigningDemo` target requires the `com.apple.developer.associated-domains` entitlement to perform cryptographic Passkey operations linked to a web domain.

| Key | Value |
| --- | --- |
| **Entitlements Path** | `TadSigningDemo/TadSigningDemo.entitlements` |
| **Associated Domains** | `webcredentials:signing.tadi.uz` |
| **Deployment Target** | iOS 16.0 |

## Associated Domains & Web Credentials

The entry `webcredentials:signing.tadi.uz` enables the **Shared Web Credentials** service — a prerequisite for Passkey operations.

### Two-Way Trust Required

1. **App → Server**: App declares the domain in `TadSigningDemo.entitlements`.
2. **Server → App**: `signing.tadi.uz` must host an AASA file at `https://signing.tadi.uz/.well-known/apple-app-site-association` listing `KXJN79YAQ7.uz.tad.TadSigningDemo` under `webcredentials`.

## Entitlements File Structure

```xml
<key>com.apple.developer.associated-domains</key>
<array>
    <string>webcredentials:signing.tadi.uz</string>
</array>
```

## Common Issues

| Code Entity | Requirement |
| --- | --- |
| `TadSigningSDK` | Requires `webcredentials` for `ASAuthorization`. |
| `signing.tadi.uz` | Must match the `rpId` in `TadSigningConfig`. |
| `uz.tad.TadSigningDemo` | Must be registered in Apple Developer Portal with Associated Domains enabled. |

> **Error 1004**: If the Bundle ID or Team ID does not match the AASA file, Passkey flow fails with "Domain not associated".
