# Privacy Permissions & App Capabilities (Info.plist)

## Privacy Permissions

| Key | Description | Purpose |
| :--- | :--- | :--- |
| `NSCameraUsageDescription` | "Required for face liveness check during biometric sign-in" | Camera frames for ONNX-based liveness detection. |
| `NSFaceIDUsageDescription` | "Used to secure your Passkey during registration and sign-in" | Required by `AuthenticationServices` for Passkey creation/assertion. |

## App Capabilities & Metadata

### Encryption Compliance
`ITSAppUsesNonExemptEncryption: false` — declares standard encryption (HTTPS/TLS only), simplifying App Store submission.

### Launch Screen
`UIColorName` set to empty string — defaults to system background color during initial UI bootstrap.

### Bundle Metadata
- **`CFBundleIdentifier`**: `uz.tad.TadSigningDemo`
- **`CFBundlePackageType`**: `APPL`
- **`CFBundleVersion`**: `1`

## Permission Trigger Logic

When `ContentView` calls `present(mode:)`:
1. **Camera**: Triggered when liveness detection UI appears (`AVFoundation`).
2. **Face ID**: Triggered when system saves or verifies a Passkey (`AuthenticationServices`).
