# Getting Started

## Environment Prerequisites

To build and run the project, ensure your development environment meets the following requirements:
*   **Xcode**: Version 16.0 or higher.
*   **iOS Device**: A physical device running iOS 16.0 or higher is required for biometric (FaceID) and camera-based liveness features.
*   **Swift**: Version 5.9.

## Repository Onboarding

### 1. Cloning the Repository
Clone the repository to your local machine. Note that the project relies on a local Swift Package for the core SDK logic.

### 2. Resolving the TadSigningSDK Dependency
The `TadSigningDemo` app consumes the `TadSigningSDK` as a local package dependency. In the provided configuration, this is referenced via a hardcoded local path.

*   **Action Required**: Update the path in `project.yml` or within Xcode project settings under `XCLocalSwiftPackageReference` to match your local directory structure.

### 3. Opening the Project
Open `TadSigningDemo.xcodeproj` to begin.

## Running the Application

### 1. Select the Scheme and Device
1.  Select the `TadSigningDemo` scheme in the Xcode toolbar.
2.  Select a physical iOS device as the run destination.
    *   *Note*: Simulator usage is restricted for liveness checks and Passkey flows that require secure hardware.

### 2. Code Signing
The project is configured with a specific development team (`KXJN79YAQ7`). Update the `DEVELOPMENT_TEAM` in Build Settings to match your own Apple Developer account.

### 3. Associated Domains
In Xcode → Signing & Capabilities, ensure **Associated Domains** is added with `webcredentials:signing.tadi.uz`.

## Key Components Reference

| Component | File Path | Role |
| :--- | :--- | :--- |
| **App Entry** | `App.swift` | Defines the SwiftUI `@main` entry point. |
| **Main UI** | `ContentView.swift` | Manages the form for credentials and SDK invocation. |
| **SDK Config** | `SDKConfig.swift` | Stores API endpoints and the ES512 public key for the SDK. |
| **Permissions** | `Info.plist` | Declares `NSCameraUsageDescription` and `NSFaceIDUsageDescription`. |
| **Entitlements** | `TadSigningDemo.entitlements` | Configures `associated-domains` for Passkey support. |

## Troubleshooting Dependency Issues
If Xcode fails to find `TadSigningSDK`:
1.  Verify the path in `project.yml`.
2.  Ensure the `Frameworks/` directory is not blocked by local file permissions (it is gitignored).
3.  Try **File → Packages → Update to Latest Package Versions**.
