# ONNX Runtime Dependency

The `TadSigningSDK` uses **ONNX Runtime (ORT)** as its core ML inference engine for real-time face liveness detection during registration and authentication.

## Dependency Specification

| Attribute | Value |
| :--- | :--- |
| **Identity** | `onnxruntime-swift-package-manager` |
| **Location** | `https://github.com/microsoft/onnxruntime-swift-package-manager` |
| **Version** | `1.24.2` |

## Role in Face Liveness Detection

Prevents unauthorized access using static images or digital screens (anti-spoof).

### Inference Pipeline
1. **Environment Init**: `ORTEnv` created to manage logging and thread pools.
2. **Session Creation**: `ORTSession` instantiated using `.onnx` model bundled in SDK resources.
3. **Data Transformation**: Camera frames converted to `ORTValue` tensors (4D: `[Batch, Channels, Height, Width]`).
4. **Inference**: `ORTSession.run()` called with input tensors.
5. **Post-processing**: Output `ORTValue` parsed to determine liveness score.

## Code Entity Mapping

| Concept | ORT Entity | Responsibility |
| :--- | :--- | :--- |
| **Runtime Context** | `ORTEnv` | Manages lifecycle of the ONNX environment. |
| **Model Instance** | `ORTSession` | Loads model and executes computation graph. |
| **Tensor Data** | `ORTValue` | Wraps Swift arrays/buffers into tensors. |
| **Execution Options** | `ORTSessionOptions` | Configures hardware acceleration (CoreML, CPU). |

ONNX Runtime `1.24.2` supports CoreML execution providers, allowing the SDK to leverage the Apple Neural Engine (ANE) for hardware-accelerated inference.
