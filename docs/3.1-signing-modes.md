# Signing Modes: Register vs. Sign

## Overview of TadSigningMode

| Mode | Enum Case | Purpose |
| :--- | :--- | :--- |
| **Registration** | `.register` | Associates a user's `bankId` with a new Passkey and captures face liveness for initial enrollment. |
| **Signing** | `.sign` | Authenticates an existing user via face liveness and Passkey signature to generate a JWT. |

## Data Flow: DTO and OTP Pattern

1. **User Input**: User provides `bankId` and optional `otp` in the SwiftUI Form.
2. **DTO Construction**: If `otp` is not empty, it is wrapped in `["otp": otp]`.
3. **SDK Injection**: Dictionary is passed to `dto` parameter of `TadSigningViewController`.

> **Key Concept**: The OTP (SMS code) is only sent **once** during `.register`. After that, `.sign` uses Face ID + Passkey — no SMS needed.

## Completion Handler & Result Types

### Success Path
- **`jwt`**: JSON Web Token (returned in `.sign` mode).
- **`requestId`**: Unique transaction identifier.

### Failure Path
- **`code`**: A `TadSigningError` enum value.
- **`message`**: Human-readable description.

## UI State Management

1. Before presenting SDK: `isLoading = true`, buttons disabled.
2. After completion: `isLoading = false`.
3. Color coding: `.green` for success, `.red` for failure.
