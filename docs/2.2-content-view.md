# Main UI & SDK Invocation (ContentView.swift)

`ContentView` is the primary user interface for the TadSigningDemo application. It provides a form-based entry point for user credentials and orchestrates the invocation of `TadSigningSDK`.

## UI Structure and State Management

Built using a SwiftUI `Form` inside a `NavigationStack`. Manages these `@State` properties:

*   **`bankId`**: User identifier (default: "demo-user-001").
*   **`otp`**: One-time password (default: "12345").
*   **`isLoading`**: Toggles `ProgressView` and disables action buttons.
*   **`result` / `resultColor`**: Displays the outcome (success/failure).

## Action Buttons

Two buttons trigger the SDK via `present(mode:)`:
1.  **Register** → `.register` mode
2.  **Sign In** → `.sign` mode

## UIKit Bridge: topViewController()

Because the SDK provides a `UIViewController` and SwiftUI `ContentView` is a `View` struct, the app must bridge the two paradigms:

| Step | Logic |
| :--- | :--- |
| **1. Find Scene** | Filter `connectedScenes` for `foregroundActive` `UIWindowScene`. |
| **2. Find Window** | Locate the `isKeyWindow` within the active scene. |
| **3. Traverse** | Iterate `presentedViewController` until top-most controller is found. |

## Result Handling

*   **`.success`**: Green — displays `requestId` (register) or `requestId` + `jwt` (sign).
*   **`.failure`**: Red — shows error `code.rawValue` and message.

## UI/Code Entity Mapping

| UI Element | Code Entity | Purpose |
| :--- | :--- | :--- |
| **Bank ID Field** | `$bankId` | User identifier for the SDK. |
| **OTP Field** | `$otp` | One-time password passed in `dto`. |
| **Register Button** | `present(mode: .register)` | Triggers registration workflow. |
| **Sign In Button** | `present(mode: .sign)` | Triggers authentication workflow. |
| **Result Text** | `Text(result)` | Displays JWT or Error details. |
